# memory/ - お嬢様邸の永続記憶層

各エージェントがコンパクションを跨いで参照する永続記憶ファイルを置くディレクトリ。

---

## ファイル構成

| ファイル | 役割 | コミット対象 |
| --- | --- | --- |
| `MEMORY.md.sample` | 永続記憶のサンプル（テンプレート） | はい |
| `README.md` | 本書 | はい |
| `MEMORY.md` | 実体。各環境で個別に運用 | **いいえ**（`.gitignore` 対象） |

---

## 使い方

### 初回セットアップ

```bash
cp memory/MEMORY.md.sample memory/MEMORY.md
```

### 内容

`MEMORY.md` には以下を記録する：

- あなたの好み・スタイル（口調、粒度、嫌うパターン）
- 各メイドの得意分野・実績
- 過去の重要決定（採用 / 却下した方針、理由）
- 運用上の注意（このプロジェクト特有の落とし穴）

### 更新タイミング

- あなたが明示的に「覚えておけ」と言ったとき
- 重大な方針決定がなされたとき
- メイドの得意・不得意が判明したとき

---

## なぜコミットしないのか

- **環境差異**: 各オペレータ（あなた）の好み・運用スタイルが異なるため、共有しても意味が薄い。
- **機密混入リスク**: 過去の決定や好みの中に、外部に出すべきでない文脈が混じる可能性がある。
- **頻繁な更新**: コミット履歴を汚さないため、ローカルでのみ管理する。

機密情報そのものを書くことは推奨されない（[SECURITY.md](../SECURITY.md) 参照）が、
万が一の保険として `.gitignore` 除外している。

---

## エージェントからの参照

各ロールの指示書（`instructions/*.md`）から `memory/MEMORY.md` を読み込む実装は今後追加予定。
現状はサンプルとして構造のみ定義。
